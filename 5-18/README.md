# 5-18 杭州路演准备

> 目标：5 月 18 日前后赴杭州参加 demo day、与 MBA 同学/前同事/前领导/潜在投资人见面，5–10 分钟讲清楚 Jason 在做什么、为什么有价值，争取问题、挑战、潜在投资。

## 目录

| 文件 | 用途 |
|------|------|
| `01-background.md` | 项目与个人背景：Jason 自述 + 已公开资料梳理（事实底稿，所有话术从这里派生） |
| `02-coach-questions-round1.md` | 教练第一轮问题：澄清模糊点、压力测试商业模式 |
| `03-coach-questions-round2.md` | 第二轮（基于 R1 答案生成） |
| `10-pitch-5min.md` | 5 分钟核心 pitch 话术（最终交付） |
| `11-pitch-10min.md` | 10 分钟扩展版 |
| `12-audience-cuts/` | 不同听众的剪裁版本（MBA 同学 / 前同事 / 投资人 / 理想主义者） |
| `20-deck-outline.md` | PPT 大纲与每页要点 |
| `30-qa-playbook.md` | Q&A 预案（最难的 20 个问题及简要回答） |
| `40-followup-template.md` | 见面后 24h 内的跟进邮件/消息模板 |

## 工作流

1. **R1**：教练抛问题 → Jason 回答 → 写入 `01-background.md` 补充段
2. **R2**：基于 R1 答案再问 → 直到背景文档"密度足够"
3. **派生话术**：从背景文档提取 5min / 10min / 受众剪裁版
4. **PPT 大纲**：每页一个故事单元，配 5min 话术
5. **Q&A 预案**：从教练问题里选最尖锐的 20 个，准备回答
6. **跟进模板**：见面后立即发，含一句话 ask + 一个轻量行动

## 原则

- **真实优先**：所有话术只能来自背景文档里的事实，不夸大、不画饼
- **不同人不同切入**：技术朋友讲架构，MBA 讲商业，理想主义者讲意义
- **永远留一个 ask**：每次结束都给对方一个明确的下一步（哪怕是"读这两篇 blog"）
