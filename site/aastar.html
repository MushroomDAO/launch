GitHub aastar.io Telegram

Why AAStar

Web3 hasn't gone mainstream because using it is hard: gas, seed phrases, chain-switching, wallet UX. AAStar 是一组**面向 builder 的 ERC-4337 + EIP-7702 基础设施组件**——每一个都 完全开源 (Apache 2.0)， 解决一个独立的 Web3 难用痛点，组合起来让任何应用都能给用户提供 Web2-grade UX。

所有合约 + 服务部署在 Sepolia 公开测试，已 Etherscan verified。Operator 角色由 SuperPaymaster v5.3.3 的 Registry 合约统一治理（GToken 质押 + Soul-Bound Token 身份）， 详见下方 生态运营者 区段。

核心组件 / Components for Builders

All Apache 2.0 license. Use freely, fork freely, no permission needed. Production licenses + commercial Pro services via HyperCapital.

AirAccount

Beta

解决：私钥太难管。用 WebAuthn / Passkey + TEE-backed BLS 把 EOA 私钥替换成 指纹/Face ID，体验等价于 Apple ID 登录。

如何用：集成 @aastar/airaccount SDK， 1 行代码替换 wagmi/ethers connector。HSM (Optee) 可选硬件后端，企业级 KMS 可挂。

airaccount airaccount-contract · Apache 2.0

SuperPaymaster

v5.3 Live Sepolia

解决：用户没 ETH 也得能用你的应用。可扩展的 ERC-4337 paymaster proxy — 多个 operator 共享一套合约，按 PNTs 计费、按 SBT 准入、按 SLA 路由。

如何用：接 paymasterAndData 走 SuperPaymaster proxy； 或自己跑一个 PaymasterV4 (单机版) 用 PaymasterFactory 部署。

SuperPaymaster Etherscan · Apache 2.0

YetAnotherAA

v0.7 / 7702

解决：ERC-4337 v0.7 + EIP-7702 双模式智能账户实现，BLS 多签签名聚合， 可执行 batched calls / sponsored ops / 自定义验证逻辑。

如何用：用 YetAnotherAA contract 作为 EOA 的 7702 delegate； 或部署成独立 smart account；可作为 AirAccountDelegate 的扩展实现。

YetAnotherAA · Apache 2.0

AAStar SDK

TS / JS

解决：把上面的合约和服务封装成 dev-friendly 一行集成 SDK。 包含 AirAccount login + SuperPaymaster billing + CometENS 子域名 + signing helpers。

如何用：npm i @aastar/sdk， 5 分钟跑通"邮箱注册 + gasless 第一笔交易"demo。

aastar-sdk · Apache 2.0

生态运营者 / Ecosystem Operators

AAStar 不只是给开发者用的库——它也是一个**多角色运营生态**。任何人都可以质押 GToken 申请下面 7 个 Operator 角色之一，运行节点 / 提供服务，赚取 PNTs 收入。所有角色 + 罚没 + 信誉 逻辑由 Registry v5.3.3 合约 统一治理。

角色 / Role 职责 最低质押 门票
GToken burn		 Stake 锁定期
KMS
Key Management Service		 企业级密钥托管 / TEE 节点。运行 OPTee 或 HSM，托管 AirAccount 用户的硬件钱包凭证。 最严苛的 SLA，最高的 stake / 罚没。 100 GT 10 GT 30 天
PAYMASTER_SUPER
SuperPaymaster Op		 运行 SuperPaymaster proxy 实例——为整个生态付 gas。需高可用、跨链支持、 低延迟 RPC、PNTs 锁定流动性。 50 GT 5 GT 30 天
PAYMASTER_AOA
Account-Owned Account		 单一账户 paymaster——为特定 dApp 或社区代付 gas（轻量、自治）。 用 PaymasterFactory 一键部署。 30 GT 3 GT 30 天
DVT
Distributed Validator		 分布式验证节点——参与 BLS 签名聚合、共识、链下计算验证。运行 DVTValidator 合约。 30 GT 3 GT 30 天
ANODE
Aggregator Node (BLS)		 BLS 签名聚合器节点。把多个 DVT 验证的签名聚合成一个，提交到 BLSAggregator 合约。降低链上 gas，提高吞吐。 20 GT 2 GT 30 天
COMMUNITY
Community Node		 社区运营方——任何能聚集 ≥50 人组织的发起人。注册社区域名 (ENS)、 在 CoS72 上运营任务/投票/积分。无最低 stake、纯门票准入。 0 30 GT —
ENDUSER
Individual User		 协议的终身用户——任何持有 ≥0.3 GToken 的个人。能用所有公共品（AirAccount、 Cos72、Sin90 等），可参与积分网络作为表达者/创作者/建设者。 0 0.3 GT —

怎么申请：在 Registry Write Contract 调 grantRole(roleId, address, stakeAmount, metadata) 即可（先在 GToken 上 approve 给 GTokenStaking）。 详细文档：SuperPaymaster repo。
