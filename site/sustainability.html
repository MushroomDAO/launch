可持续规划 / Sustainability Plan

数字公共物品不是永远免费、永远亏损的代名词——而是需要多元可持续的资金来源。我们不依赖单一路径：Grant、数字公共物品冷启动、协议费、增值服务共同构成一个有韧性的生态支撑体系。 *注意：以下是社区的可持续规划，包括但未区分Mycelium社区和双生商业公司，需要实践中确定。*

可持续的多元结构

🌱

Phase 1：数字公共物品冷启动

Cold Launch 天使赞助 + Grant 申请 + Gitcoin Grants / Optimism RetroPGF。非稀释性资金，证明交付能力，建立信任。

⚙️

Phase 2：协议自造血

基础设施协议费 + 社区增值服务产生现金流。微小但稳定，随规模自然增长，覆盖日常运营。

🚀

Phase 3：生态飞轮

社区 AI 订阅 + 协议生态伙伴 + DAO 国库进入正向飞轮，实现真正的长期可持续。

我们不追求 VC 融资驱动的"高速增长然后退出"模式。目标是一个可持续运营 20 年的协议网络——像 Linux 一样，即使创始人消失，生态还在运转。

开源数字公共物品的可持续案例

我们研究了历史上成功的开源商业化路径，提取对 Mycelium Protocol 有借鉴价值的模式：

Open Core Red Hat Linux

Linux 内核永久免费开源；Red Hat 销售的是企业版认证、长期支持、安全补丁、培训和咨询服务。2019年以 340 亿美元被 IBM 收购。

核心逻辑：
企业愿意为"稳定性保证"付费，而不是为软件本身付费。代码免费，信任和服务值钱。
→ 我们的对应：Cos72/Sin90 开源，托管版 + 企业定制版收费
Freemium SaaS GitLab

社区版完全开源，可自托管。GitLab.com 的 SaaS 托管版提供免费基础功能，付费解锁 CI/CD 高级、安全扫描、企业 SSO 等。年收入超过 5 亿美元。

核心逻辑：
免费版获客，付费版变现。托管 = 省心，增值模块 = 提效，两者都值钱。
→ 我们的对应：社区自托管免费，mushroom.cv 托管版按规模收费
协议费 Uniswap Protocol

智能合约代码完全开源，任何人可 fork。Uniswap Labs 开启协议费（0.15%），仅收取通过官方前端发生的交易。DAO 国库持有大量 UNI Token。

核心逻辑：
协议使用量产生的微小费率，乘以足够大的交易量，形成稳定现金流。开源不代表放弃协议费。
→ 我们的对应：SuperPaymaster 协议费 min(0.001 ETH, 10%)
基金会 + 赠款 Wikimedia Foundation

Wikipedia 内容完全免费，无广告。通过每年一次的全球募捐，年收入超过 1.5 亿美元。捐款人感受到"这个东西值得存在"的情感驱动。

核心逻辑：
当一个数字公共物品真正被人依赖，会有足够多的人愿意自愿支持它存在。规模和信任是前提。
→ 我们的对应：Cold Launch Grant 赞助 + Gitcoin/Protocol Guild 类赠款申请
Token 经济 Protocol Labs (IPFS/Filecoin)

IPFS 协议开源免费；Filecoin 通过 Token 激励存储提供者，形成去中心化存储网络。协议基金会持有部分 Token，通过赠款支持生态开发。

核心逻辑：
Token 不是投机工具，而是网络参与的激励机制。参与即贡献，贡献即获益，形成自强化网络效应。
→ 我们的对应：Governance Token + 社区积分（贡献兑换服务使用权）
二次方融资 Gitcoin Grants

通过二次方融资（Quadratic Funding），小额众多捐款被算法放大。以太坊基金会等机构提供配套资金。已为 3,000+ 开源项目分发超过 5,000 万美元。

核心逻辑：
民主化的公共物品融资——支持者数量比单笔金额更重要，鼓励广泛社区参与。
→ 我们的对应：申请 Gitcoin Grants、Optimism RetroPGF、以太坊基金会赠款

开源协议与双生商业公司 / License & Twin Company

代码开放给所有人，商业化由社区授权的唯一合作伙伴承载，收益回馈公共物品可持续发展。

开源协议：Apache 2.0

我们的所有代码、协议、工具均为数字公共物品。采用 Apache 2.0 开源协议：

个人用户

完全免费。clone、部署、研究、二次开发、自用，无任何限制。这些产品就是为你做的。

小社区（≤50人）

完全免费，无需授权（自声明 ≤50 成员）。自由使用、fork、开发、部署。鼓励注册但非强制。

大型组织（>50人）

非商业用途依然免费！仅需轻量审核（非付费墙）——7天内保证答复。审核通过即登记，公开透明。若判定为商业性质，建议联系商业授权。

商业公司

禁止将本项目代码直接或包装后作为商业服务出售。商业用途需通过唯一授权合作伙伴 HyperCapital（jason@aastar.io）。

Apache 2.0 允许自由使用、修改和分发，保留署名即可。 我们帮助需要帮助的人，不是免费帮助商业公司赚钱，商业合作走商业渠道即可。

我们采用 OSI 认证的 Apache 2.0 开源协议 —— 这里的"开源"是 Open Source Initiative 定义的、具有法律意义的真正开源，而非宽泛的市场用语。

FAQ：信任、透明与治理

Q1: 创始人同时创立社区和公司，会不会"左手倒右手"、贪污或滥权？

链上全透明：社区所有资金存放在 Gnosis Safe 多签钱包，这是第三方托管，非任何个人可单独操作。所有交易历史任何人可查。

ENS 实名：所有相关钱包地址均有 ENS 记录，关联到具体的个人或公司员工身份。链上记录不可篡改、不可抵赖。

委员会决策：社区国库的每一笔支出都经过社区委员会商议决策，不是任何一个人说了算。国库会持续维护公开的支出列表。

商业合规约束：商业公司受工商、税务、法律的完整约束。公司与 DAO 组织签署的法律协议对公司具有法律约束力。

Q2: 为什么只能用指定的商业公司（HyperCapital），不能用其他服务商？

公共物品的经济基础：HyperCapital 承诺 40% 收益回流社区国库。如果允许任意商业公司进入而不回馈，社区就失去了可持续的经济来源。排他性不是垄断，而是保护公共物品的"生态税"。

类比：就像 Linux 开源社区与 Red Hat 的关系——代码完全开放，但专业商业服务由获得授权的商业实体提供，收益反哺开源社区。

授权有条件：独家授权设定期限与绩效条件，社区治理保留审核权。如果商业公司不能履约，社区可以收回授权。

Q3: 公司持有 20% GToken，会不会操控社区治理？

20% 远低于绝对控制所需的 51%。GToken 是公开销售的治理代币，随着更多人参与，公司持有比例会被自然稀释。

公司持有 GToken 的目的是激励对齐：公司做得越好 → 生态越繁荣 → token 越有影响力 → 公司自身获益。这是利益绑定，不是权力集中。

Q4: 社区为什么不持有公司股权？

社区是一群人的组织，持有公司股权可能拖累商业决策效率。社区需要的是自由的氛围和充足的资金，而不是股东席位。

社区获得的是：40% 商业收益（法律协议保障）+ 三年内分阶段注入运营资金。拿钱做想做的事，但有大的方向约束（三大产品线、三大协议）。

Q5: 即便有人想做坏事，有什么制约？

多重制衡：商业行为受商业法律约束（税务、工商、合同）；社区行为受链上透明和集体决策约束。决策流程化，不是一个人说了算。

创始人也不一定是持有 token 最多的人——GToken 是公开销售的，面向所有需要帮助的人提供。

所有代码开源，所有资金流向可查，所有决策有记录。透明本身就是最好的防腐剂。

Q6: 这个模式追求的平衡是什么？

每个人都要生存，社区发展需要经济支持。商业公司提供这种支持——从商业中获得帮助，让社区触及商业无法到达的领域：建立信任、服务个体、帮助非营利社区。

所有代码对个人和小社区（≤50人）完全免费、无需授权。大型组织申请免费的社区使用授权。需要商业服务？联系唯一授权的合作伙伴即可。

不是免费帮助商业公司赚钱，商业合作走商业渠道即可。
