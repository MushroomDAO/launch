# AG-08 社区答疑 Agent

> **一句话**：给 DAO / 社区 / 开源项目部署一个永不下线的答疑机器人，把散落在文档、公告、讨论里的知识整合起来，让新人不再重复问同样的问题。

---

## 目标用户

- DAO / Web3 社区：代币经济、治理规则、提案流程
- 开源项目社区：安装文档、常见报错、贡献指南
- 在线教育社区：课程答疑、作业辅导
- 行业协会 / NGO：政策解读、活动通知、成员服务

## 核心痛点

| 痛点 | 现状 | Agent 解决后 |
|------|------|-------------|
| 新人同样的问题问 100 遍 | 老人疲于回答，社区体验差 | 知识库自动回答 |
| 文档分散在多处 | Notion / GitHub / Discord 各一份 | 统一索引，一个入口 |
| 夜间无人值守 | 亚洲用户 vs 欧美运营，时区错位 | 7×24 自动答疑 |
| 活动通知触达率低 | 公告被刷屏淹没 | 主动 @ 相关成员 + 摘要推送 |

---

## 功能清单（v1）

### 答疑
- 基于知识库（Docs / FAQ / 公告历史）回答问题
- 回答时附带来源链接（"来自：[治理文档第3节]"）
- 识别无法回答的问题 → 自动 @ 对应维护者 / 创建 issue

### 知识库管理
- 从 Notion / GitHub Wiki / Gitbook 同步（定时抓取）
- 新公告发布 → 自动更新知识库
- 支持手动上传 PDF / Markdown

### 社区洞察（运营视角）
- 统计本周 Top 10 被问问题 → 帮运营识别文档盲点
- 识别高频抱怨 → 生成问题报告给维护团队

### 平台接入
- Telegram Bot
- Discord Bot
- 网页内嵌 Widget（给官网用）

## 技术方案

```
用户在 Telegram/Discord/网页 提问
    │
    ▼
消息接收 → 意图分类
  ├─ 常规问答 → RAG 查知识库 → LLM 回答
  ├─ 操作引导 → 推送步骤链接
  └─ 人工升级 → @ 维护者 + 记录问题
    │
    ▼
知识库更新（定时任务）
  Notion API / GitHub API → 增量抓取 → 重新向量化
```

## DAO 透明治理特殊功能

针对 Web3 / DAO 场景的额外能力：

| 功能 | 说明 |
|------|------|
| 提案追踪 | 监听链上治理合约，自动播报新提案 + 投票截止提醒 |
| 资金流向播报 | Gnosis Safe 有新 tx → 自动推送摘要到社区频道 |
| 角色申请向导 | 引导用户完成 GToken 购买 → 角色注册全流程 |
| 贡献记录查询 | "我的贡献积分是多少" → 查链上/链下记录 |
