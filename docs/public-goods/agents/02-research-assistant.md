# AG-02 研究助手 Agent

> **一句话**：把一堆文件扔给它，告诉它你要什么——它帮你提炼、对比、生成报告，像一个随时在线的研究员。

---

## 目标用户

- 研究生 / 独立研究者：文献综述、笔记整理
- 律师 / 顾问：合同/案例快速阅读、关键信息提取
- 投资人 / 分析师：多份 PDF 年报/白皮书比较
- 内容创作者：素材整理 → 输出大纲
- 任何需要"把一堆资料变成有用信息"的人

## 核心痛点

| 痛点 | 现状 | Agent 解决后 |
|------|------|-------------|
| 文献/文件太多，读不完 | 手动读、做笔记 | 上传 → 自动摘要 + 关键词 |
| 多文件对比困难 | 开多个标签页反复翻 | 一条指令 → 对比表格 |
| 从资料到报告耗时 | 写初稿要数小时 | 给结构 → Agent 填内容 + 引用出处 |
| 找不到某个具体信息 | 全文 Ctrl+F | 自然语言问 → 精准定位 + 原文引用 |

---

## 核心指令集（v1）

用户用自然语言发指令，Agent 理解意图并执行：

| 指令类型 | 示例输入 | 输出 |
|---------|---------|------|
| **摘要** | "帮我总结这 5 篇文章的核心观点" | 每篇 3-5 句摘要 |
| **提取** | "提取所有文件里提到的数据和数字" | 结构化表格 |
| **对比** | "这 3 份方案在成本/时间/风险上有什么差异" | 对比矩阵 |
| **问答** | "第 2 篇文件里关于监管风险说了什么" | 原文定位 + 引用段落 |
| **大纲** | "基于这些资料帮我生成一份研究报告大纲" | 带一级/二级标题的大纲 |
| **报告** | "按这个大纲填写内容，每个要点引用来源" | 带脚注的草稿报告 |
| **时间线** | "把这些事件按时间顺序整理" | 时间轴 Markdown |

## 工作流程

```
用户上传文件（PDF / Word / TXT / MD / 网页URL）
    │
    ▼
文档解析 → 切片（chunk size: 512 tokens, overlap: 50）
    │
    ▼
向量化 → 存入本地向量库（按会话隔离）
    │
    ▼
用户发指令
    │
    ├─ 全局指令（摘要/大纲/报告）──► 遍历所有文档，LLM 聚合
    │
    └─ 检索指令（问答/提取）──► RAG 检索最相关 chunk → LLM 回答 + 原文引用
    │
    ▼
输出 Markdown / 可下载 Word / PDF
```

---

## 支持的输入格式

| 格式 | 说明 |
|------|------|
| PDF | 支持扫描件（OCR）|
| Word (.docx) | 保留段落结构 |
| 纯文本 / Markdown | 直接处理 |
| 网页 URL | 自动抓取正文（去除导航/广告）|
| YouTube / 视频链接 | 提取字幕文本（需字幕存在）|

---

## 输出格式

- **Markdown**（默认）：可直接粘贴到 Notion / 飞书 / Obsidian
- **Word (.docx)**：一键下载，用于提交或分发
- **JSON**（结构化提取结果，供其他系统消费）

---

## 技术方案

```
Web UI（拖拽上传 / 聊天框）
    │
    ▼
Agent Server（Python FastAPI）
    ├─ 文档解析：PyMuPDF / python-docx / trafilatura
    ├─ 切片 + 向量化：LangChain Text Splitter + OpenAI/Claude Embeddings
    ├─ 向量库：Chroma（本地）
    ├─ LLM：Claude Sonnet（高质量研究任务）/ Haiku（快速预览）
    └─ 输出渲染：Markdown → python-docx / WeasyPrint
```

---

## 会话模式 vs 项目模式

| 模式 | 说明 | 适合场景 |
|------|------|---------|
| **会话模式** | 上传 → 问 → 关闭即清除 | 临时一次性任务 |
| **项目模式** | 文件持久化存储，可随时追问 | 长期研究项目 |

项目模式下，向量库按项目 ID 隔离，用户可管理多个研究项目。

---

## 隐私设计

- 所有文件和向量数据默认存本地，不上传第三方
- 仅 LLM 调用时发送相关 chunk 文本（非整份文件）
- 用户可选择完全离线（本地 Ollama 模型）

---

## 成本估算（月）

| 用法 | 文档量 | LLM 成本 | 合计 |
|------|--------|---------|------|

| 中度（研究生日常）| ~200 份 | ~¥80 | **~¥80/月** |
