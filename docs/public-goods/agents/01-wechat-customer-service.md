# AG-01 微信客服 Agent

> **一句话**：给任何中小商家/服务商，部署一个 7×24 能答问、能引导流程、能调工具的微信客服。

---

## 目标用户

- 有微信公众号 / 企业微信的中小商家
- 客服消息量 < 500 条/天、没有专职客服或只有 1 人兼职
- 典型行业：餐饮外卖、美容美发、本地零售、在线教育、小型 SaaS

## 核心痛点

| 痛点 | 现状 | Agent 解决后 |
|------|------|-------------|
| 下班后无人回复 | 用户等到次日，流失 | 秒回，7×24 |
| 重复问题耗人力 | 同一问题每天问 N 次 | RAG 知识库一次配置 |
| 人工引导购买低效 | 客服判断力参差 | 规则触发 → 购买 / 预约流程 |
| 售后信息查不到 | 客服要翻系统 | 接 CRM/订单工具自动查 |

---

## 功能边界（v1 范围）

**做的：**
- 基于知识库（产品介绍 / FAQ / 价格表）回答问题
- 识别关键词 → 推送引导链接（购买页 / 预约页 / 表单）
- 多轮对话保持上下文（单次会话内）
- 无法回答时 → 转人工 + 记录问题

**不做（v1）：**
- 主动发消息 / 营销推送
- 跨会话记忆（需 CRM 集成，v2）
- 语音 / 图片识别（v2）
- 接订单系统自动查物流（v2，工具扩展）

## 对话流程图

```
用户发消息
    │
    ▼
关键词预筛（价格 / 怎么买 / 退款…）
    │
    ├─ 命中规则 ──► 直接返回模板 + 引导链接
    │
    └─ 未命中 ──► LLM + RAG 查知识库
                      │
                      ├─ 置信度 ≥ 0.7 ──► 回复用户
                      │
                      └─ 置信度 < 0.7 ──► "稍等，我帮你转接人工" + 记录
```

## 知识库结构（商家配置）

商家只需上传以下文件（支持 PDF / Word / 纯文本）：

```
knowledge/
├── products.md        # 产品/服务列表 + 价格
├── faq.md             # 常见问答（至少 20 条）
├── policies.md        # 退换货 / 售后政策
└── flows.md           # 流程触发规则（关键词 → 链接映射）
```

系统自动切片 → 向量化 → 存入本地 Chroma。

---

## 工具调用（可选扩展）

| 工具 | 触发场景 | 说明 |
|------|---------|------|
| `query_order(order_id)` | 用户说"查我的订单" | 接商家 ERP/电商后台 API |
| `create_ticket(content)` | 问题无法解答 | 创建售后工单，通知人工 |
| `send_link(scene)` | 识别购买意向 | 推送购买页 / 预约表单 URL |
| `query_inventory(sku)` | "XX 还有货吗" | 查库存系统 |

工具以 MCP server 形式接入，商家自选开启。

## 技术方案

```
微信公众号 / 企业微信  ──Webhook──►  Agent Server
                                        │
                          ┌─────────────┼─────────────┐
                          ▼             ▼             ▼
                      关键词引擎    LLM + RAG       工具调用
                      (本地规则)   (Claude API     (MCP Tools)
                                  / Ollama)
                          │
                          ▼
                      回复消息 → 微信平台 → 用户
```

- LLM：Claude Haiku（低延迟低成本）或本地 Qwen2.5-7B（完全离线）
- 向量库：Chroma（本地）或 LanceDB（Serverless）
- 部署：Docker 单容器，一台 2核4G 云主机即可

---

## 配置示例

```yaml
# config.yaml（商家配置文件）
business_name: "小明奶茶"
language: zh
llm:
  provider: claude          # 或 ollama
  model: claude-haiku-4-5
  temperature: 0.3

knowledge_dir: ./knowledge
vector_store: chroma

handoff_threshold: 0.65    # 低于此置信度转人工
handoff_message: "正在为您转接人工客服，请稍候～"

flows:
  - keywords: ["购买", "下单", "怎么买"]
    action: send_link
    url: "https://shop.xiaomingnacha.com/order"
  - keywords: ["退款", "不想要了"]
    action: send_link
    url: "https://shop.xiaomingnacha.com/refund"
```

## 上线步骤（商家视角）

```
1. 注册微信公众号 / 企业微信（已有则跳过）
2. 下载 AG-01 Docker 镜像，填写 config.yaml
3. 上传知识文档到 knowledge/ 目录
4. 运行 docker compose up
5. 在微信后台填写 Webhook URL（服务器地址）
6. 发测试消息验证 → 上线
```
