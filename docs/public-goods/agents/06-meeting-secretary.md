# AG-06 会议秘书 Agent

> **一句话**：录音或文字丢给它，它出纪要、提行动项、写跟进邮件——让会议结束时就有可执行的下一步。

---

## 目标用户

- 创业团队周会、评审会
- 客户访谈 / 用户调研记录
- 社区 / DAO 线上会议（腾讯会议 / Zoom）
- 独立工作者的客户沟通记录

## 核心痛点

| 痛点 | 现状 | Agent 解决后 |
|------|------|-------------|
| 开完会没人整理纪要 | 靠记忆或手动整理，经常漏 | 上传录音 → 自动出纪要 |
| 行动项不清晰 | 讨论了很多，不知道谁负责什么 | 自动提取：人 + 事 + 时间 |
| 访谈内容散乱 | 用户说了很多，要点埋在里面 | 自动提炼关键洞察 |
| 跟进邮件要写很久 | 会后发邮件确认，每次都要想 | 一键生成跟进邮件草稿 |

---

## 核心功能（v1）

### 输入
- 音频文件（MP3 / WAV / M4A / 微信语音 .amr）
- 视频文件（MP4 — 提取音轨）
- 腾讯会议 / Zoom 自动字幕导出文件（.txt / .vtt）
- 直接粘贴文字记录

### 输出
1. **会议纪要**：时间 / 参会人 / 议题 / 结论
2. **行动项清单**：责任人 + 事项 + 截止日期
3. **关键决策记录**：本次会议做出的核心决策
4. **跟进邮件草稿**：英文 / 中文可选
5. **用户访谈洞察**（研究模式）：关键需求 / 痛点 / 引用原话

## 技术方案

```
音频 / 视频 / 文字
    │
    ▼
语音转文字（如需）
  ├─ 本地：Whisper（medium 模型，中文效果好）
  ├─ 本地：FunASR（阿里，中文更优）
  └─ 云 API：腾讯云 ASR / 阿里云 ASR
    │
    ▼
说话人分离（可选）
  Whisper + pyannote（区分"张三说"/"李四说"）
    │
    ▼
LLM 处理
  ├─ 纪要生成（结构化摘要）
  ├─ 行动项提取（命名实体 + 时间识别）
  └─ 邮件草稿生成
    │
    ▼
Markdown 输出 + 可选同步到 Notion / 飞书
```

---

## 说话人识别（准确度说明）

- 能区分不同声音，但不能自动知道名字
- 标记为"说话人 A / B / C"，用户最后对应标注姓名
- 多次会议后，可学习声纹（v2）

## 成本估算（月）

| 方案 | 每次会议成本 | 说明 |
|------|------------|------|
| 本地 Whisper + 本地 LLM | ¥0 | 需要一台性能还可以的电脑 |
| 本地 Whisper + Claude API | ~¥1-3/次 | 1小时会议约消耗 5K tokens |
| 全云（腾讯 ASR + Claude）| ~¥3-8/次 | 无需本地配置 |
